Logistics Operations Analytics Suite

This is a multi-deliverable analytics project that simulates the role of a data & BI analyst embedded within a logistics company. The focus is on translating raw operational data into structured, usable insights that directly support business functions.

The dataset covers a fictional logistics company operating across the United States, containing 14 relational tables spanning fleet operations, driver performance, customer accounts, route data, fuel purchases, maintenance records, and safety incidents; covering three years of operational data from 2022 to 2024.

Data source: Kaggle: Logistics Operations Database

SQL Server Power BI Python Data Warehousing Statistics Operations Research

Database & Warehouse Design

The source data was imported into a relational SQL Server database (OLTP), 14 tables reflecting the company's day-to-day operational system. Analytical queries on an OLTP database are slow and complex, requiring many joins. A separate analytical warehouse (OLAP) was designed using a star schema, the industry standard for business intelligence. Three data marts were built within the warehouse (one per department), each with its own fact table and a set of surrounding dimension tables.

Six conformed dimensions are shared across all marts: DimDate, DimCustomer, DimRoute, DimDriver, DimSafetyIncident, and DimLoad. Conformed dimensions are defined once and reused across multiple fact tables, ensuring consistent definitions across departments and enabling cross-functional reporting.

SQL Analysis

Five business queries were written against the warehouse to validate the design and extract initial insights. One query is shown below, the maintenance cost per mile analysis, which joins three tables across two schemas to rank trucks by operational cost efficiency.

Fleet: Maintenance cost per mile by truck

SELECT TOP 5 odt.truck_id, SUM(ofm.total_cost) / SUM(oft.actual_distance_miles) AS cost_per_mile FROM Operations.FactMaintenance ofm JOIN Operations.FactTrip oft ON oft.truck_key = ofm.truck_key JOIN Operations.DimTruck odt ON odt.truck_key = ofm.truck_key GROUP BY odt.truck_id ORDER BY cost_per_mile DESC;
