Insurance FAQ Assistant

Project Description

The Problem

Insurance companies manage vast amounts of product data, policy rules, and general procedures across multiple spreadsheets. For customers or employees, finding specific answers manually within these tables is slow and leads to communication inefficiencies.

The Solution

I developed a functional chatbot application that transforms static Excel datasets into an interactive conversation. The system uses a multi-level decision tree to guide users through categories. When a user makes a selection, the script performs a real-time data lookup in a Pandas DataFrame to retrieve the exact answer mapped to that specific product or category.

Key Deliverables

  • Interactive Streamlit Interface: A clean, button-based chat window that guides users through a multi-level decision tree.
  • Dynamic Knowledge Base: The system is fully customizable, updating the underlying Excel file automatically updates the chatbot’s responses without any code changes.
  • Automated Data Retrieval: Uses Pandas to perform real-time filtering of dataframes to match user selections with specific answers.

Technical Tools & Future Growth

Tools Used

  • Python: The core programming language used for the logic and state management.
  • Streamlit: Used to build the web-ready user interface and session handling.
  • Pandas: Employed for structured data manipulation and Excel integration.
  • Excel: Serves as the easily maintainable "database" for all FAQ content.

Future Scalability

  • SQL Migration: While currently powered by Excel, the logic is designed for a seamless transition to a SQL database to support larger enterprise datasets and multi-user environments.
  • Web Deployment: Because it is built with Streamlit, the assistant is ready to be hosted on a company website or a secure user portal.
